Gemini Multi-Prompt TTS Player Deployment Guide

This document matches the current production architecture:
- Frontend and backend run in one Cloud Run service
- Firestore stores schedules, runs, and global rate-limit counters
- Cloud Storage stores scheduled WAV and JSON artifacts
- Cloud Scheduler triggers due-schedule polling

========================================================================
1. Required GCP Resources
========================================================================

- Project ID: your real GCP project ID
- Region: recommended `us-central1`
- Cloud Run service: `gemini-multi-prompt-tts-player`
- Cloud Scheduler job: `schedule-runner`
- Cloud Storage bucket: private bucket for WAV and JSON artifacts

========================================================================
2. Required Secrets
========================================================================

Create or update these secrets in Secret Manager:

- `GEMINI_API_KEY`
- `APP_ADMIN_PASSWORD`
- `SCHEDULER_SHARED_SECRET`
- `ADMIN_SESSION_SECRET`

`ADMIN_SESSION_SECRET` is strongly recommended. The app can fall back to
`APP_ADMIN_PASSWORD`, but that is only for compatibility and should not be the
long-term production setup.

========================================================================
3. Required Environment Variables
========================================================================

These are the current runtime env vars expected by the app:

- `GCS_BUCKET_NAME`
- `APP_DEFAULT_TIMEZONE`
- `FIRESTORE_COLLECTION_SCHEDULES`
- `FIRESTORE_COLLECTION_RUNS`
- `FIRESTORE_COLLECTION_RATE_LIMITS`
- `CLOUD_SCHEDULER_LOCATION`
- `CLOUD_SCHEDULER_JOB_NAME`
- `ALLOWED_ORIGINS`

Recommended values:

- `APP_DEFAULT_TIMEZONE=Europe/Paris`
- `FIRESTORE_COLLECTION_SCHEDULES=schedules`
- `FIRESTORE_COLLECTION_RUNS=schedule_runs`
- `FIRESTORE_COLLECTION_RATE_LIMITS=rate_limits`
- `CLOUD_SCHEDULER_LOCATION=us-central1`
- `CLOUD_SCHEDULER_JOB_NAME=schedule-runner`

`ALLOWED_ORIGINS` must be a comma-separated allowlist of browser origins that
are allowed to submit state-changing requests. Example:

`ALLOWED_ORIGINS=https://gemini-multi-prompt-tts-player-7kce3noezq-uc.a.run.app,https://gemini-multi-prompt-tts-player-305416593596.us-central1.run.app`

========================================================================
4. Firestore Notes
========================================================================

Collections used by the app:

- `schedules`
- `schedule_runs`
- `rate_limits`

You also need the checked-in composite index definition:

- `firestore.indexes.json`

========================================================================
5. Cloud Run Deploy Command
========================================================================

Example deploy command:

gcloud run deploy gemini-multi-prompt-tts-player ^
  --source . ^
  --region us-central1 ^
  --allow-unauthenticated ^
  --service-account gemini-tts-sa@YOUR_PROJECT_ID.iam.gserviceaccount.com ^
  --set-secrets GEMINI_API_KEY=GEMINI_API_KEY:latest ^
  --set-secrets APP_ADMIN_PASSWORD=APP_ADMIN_PASSWORD:latest ^
  --set-secrets SCHEDULER_SHARED_SECRET=SCHEDULER_SHARED_SECRET:latest ^
  --set-secrets ADMIN_SESSION_SECRET=ADMIN_SESSION_SECRET:latest ^
  --set-env-vars "^@^GCS_BUCKET_NAME=YOUR_BUCKET_NAME@APP_DEFAULT_TIMEZONE=Europe/Paris@FIRESTORE_COLLECTION_SCHEDULES=schedules@FIRESTORE_COLLECTION_RUNS=schedule_runs@FIRESTORE_COLLECTION_RATE_LIMITS=rate_limits@CLOUD_SCHEDULER_LOCATION=us-central1@CLOUD_SCHEDULER_JOB_NAME=schedule-runner@ALLOWED_ORIGINS=https://YOUR_PRIMARY_URL,https://YOUR_SECONDARY_URL" ^
  --min-instances 1 ^
  --max-instances 10 ^
  --concurrency 80 ^
  --timeout 300 ^
  --port 8080

Notes:

- Do not set `PORT` in `--set-env-vars`
- `^@^` is used so `ALLOWED_ORIGINS` can safely contain commas
- `min-instances=1` reduces cold starts
- `max-instances=10` is the current scale setting used to handle concurrent scheduled work

========================================================================
6. Scheduler Job
========================================================================

Current internal polling endpoint:

- `POST /api/internal/run-due-schedules`

Required header:

- `X-Internal-Key: YOUR_SCHEDULER_SHARED_SECRET`

Example job creation:

gcloud scheduler jobs create http schedule-runner ^
  --location=us-central1 ^
  --schedule="0 * * * *" ^
  --time-zone="Europe/Paris" ^
  --uri="https://YOUR_CLOUD_RUN_URL/api/internal/run-due-schedules" ^
  --http-method=POST ^
  --headers="X-Internal-Key=YOUR_SCHEDULER_SHARED_SECRET,Content-Type=application/json" ^
  --message-body="{}"

========================================================================
7. Security Notes
========================================================================

Current production hardening in the app:

- Admin auth uses signed `HttpOnly` cookies
- Browser write endpoints enforce same-origin CSRF checks via `Origin`/`Referer`
- `/api/artifacts/...` requires admin auth
- Artifact path traversal is blocked in the storage layer
- Daily global rate limits are stored in Firestore:
  - login: 100/day
  - text: 200/day
  - tts: 200/day

Limitations:

- If Firestore is unavailable, the rate limiter falls back to local memory
- That fallback is only for degraded operation, not the normal production path

========================================================================
8. Player Behavior
========================================================================

Current player behavior for scheduled runs:

- Scheduled audio is prefetched from top to bottom
- Prefetch is serial: one item at a time
- Each item shows a clear state:
  - `Queued`
  - `Downloading audio...`
  - `Decoding audio...`
  - `Cached locally`
  - `Failed`
- Cached scheduled audio and text are persisted locally in IndexedDB until manually cleared
